## 🔴 嚴重問題

### 1. 缺少聯盟行銷揭露聲明（法規合規）
- **影響頁面：** 全部 5 個語言版本
- **問題：** 頁面中含有 books.com.tw 聯盟連結及 Luna Yoga Music 推薦，但未提供明顯的贊助/聯盟關係揭露聲明。依 FTC 規定及各地廣告法規，此類內容須明確標示。
- **建議修正：** 在推薦閱讀與 Luna Yoga Music 區塊附近加入小字說明，例如：
  - ZH：「*本頁包含聯盟連結，點擊購買可能為本站帶來少量佣金*」
  - EN：`*This page contains affiliate links. We may earn a commission at no extra cost to you.*`

---

### 2. 角色圖片缺少 `loading="lazy"` 屬性（效能）
- **影響頁面：** 全部 5 個語言版本（首頁角色卡片區）
- **問題：** 5 張角色 WebP 圖片（57–121 KB 各）在頁面載入時全部同步載入，即使使用者尚未捲動到角色區段，仍消耗頻寬，拖慢首屏速度。
- **建議修正：** 在每個 `<img class="char-img">` 標籤加入 `loading="lazy"`：
  ```html
  <img class="char-img" src="/images/characters/iris.webp" alt="Iris" loading="lazy">
  ```

## 🟡 重要問題

### 3. 各語言 HTML 內嵌 CSS 大量重複（效能 / 維護性）
- **影響頁面：** 全部 5 個語言版本
- **問題：** 每個語言版本的 `index.html` 都內嵌了完整的 CSS（約 10–15 KB），五個檔案幾乎完全相同，造成 40–50 KB 的重複資料。改版時需同步修改 5 個地方，容易造成不一致。
- **建議修正：** 將語言無關的通用 CSS 移至 `shared.css`，各語言 HTML 只保留語言特定的微調樣式（如字型選擇）。估計可將每頁節省 10–12 KB。

---

### 4. JA / KO / ES 缺少語言專屬 OG 封面圖（社群分享 SEO）
- **影響頁面：** ja/index.html、ko/index.html、es/index.html
- **問題：** EN 版使用 `og-cover-en.jpg`（語言專屬），但 JA / KO / ES 仍使用通用的 `og-cover.jpg`，分享到 LINE、Twitter、Facebook 時顯示與語言不符的封面，降低點擊率。
- **建議修正：** 製作對應的 `og-cover-ja.jpg`、`og-cover-ko.jpg`、`og-cover-es.jpg`，更新各語言頁面的 `<meta property="og:image">` 標籤。

---

### 5. 角色圖片 alt 文字過於簡短（SEO / 無障礙）
- **影響頁面：** 全部 5 個語言版本
- **問題：** 目前角色圖片 alt 僅有名字（如 `alt="Iris"`），無法讓螢幕閱讀器使用者或搜尋引擎了解圖片的愛的語言關聯。
- **建議修正：** 擴充 alt 文字，例如：
  - `alt="Iris，晨曦詩人 — 肯定的言語守護靈"` (ZH)
  - `alt="Iris, the Morning Poet — Words of Affirmation Guardian"` (EN)

---

### 6. JSON-LD 貨幣設定不一致（國際化）
- **影響頁面：** EN / JA / KO / ES 版本
- **問題：** 所有版本的 JSON-LD 結構化資料均設定 `"priceCurrency": "TWD"`，但 EN / JA / KO / ES 使用者期待看到 USD / JPY / KRW / MXN 等本地貨幣，可能影響搜尋引擎的語意理解。
- **建議修正：** 依語言設定對應貨幣，或如服務免費可移除 `offers` 區塊。

---

### 7. 缺少「跳至主要內容」連結（鍵盤無障礙）
- **影響頁面：** 全部 5 個語言版本
- **問題：** 鍵盤使用者每次進入頁面需 Tab 過導覽列才能到達測驗，不符合 WCAG 2.1 最佳實踐。
- **建議修正：** 在 `<body>` 最頂端加入隱藏式跳轉連結：
  ```html
  <a href="#quiz-wrapper" class="skip-link">跳至測驗內容</a>
  ```
  搭配 CSS 在 focus 時顯示。

## 🟢 建議優化項目

### 8. 增加社群信任證明（轉換率）
- **問題：** 頁面無「已有 X 人完成測驗」等社群數據，缺乏 FOMO 驅動。
- **建議：** 在英雄區或測驗按鈕下方加入統計數字（例如「累計 50,000+ 人已發現自己的守護靈」）。

### 9. 聯盟資源偏少，只有 1 本書（營收優化）
- **問題：** 每個測驗結果僅推薦 1 本書，錯失追加銷售機會。
- **建議：** 每種愛的語言增加 2–3 項推薦（可包含課程、工作手冊、練習卡片），並做 A/B 測試比較點擊率。

### 10. Language Switcher 缺少當前語言視覺標記
- **問題：** 切換語言下拉選單沒有標示「目前語言」，使用者需靠 URL 判斷自己在哪個版本。
- **建議：** 在當前語言選項前加入 ✓ 或粗體標示。

### 11. 結果頁缺少再次分享 / 深度內容引導
- **問題：** 結果頁末尾可進一步引導使用者到理論頁面或 Luna Yoga Music 播放清單，目前動線到分享後即結束。
- **建議：** 加入「了解更多關於 [愛的語言類型] 的文章 →」指向 `/theory/` 或相關資源頁。

### 12. Luna Yoga Music 區塊未標示為推薦/合作
- **問題：** Luna Yoga Music 卡片視覺上與推薦閱讀並列，但未標示是否為合作/官方推薦，使用者可能感到困惑。
- **建議：** 加入小標籤如「🎵 官方合作頻道」或「♡ 我們的推薦」。

### 13. 測驗結果頁缺少各愛的語言人口分布數據
- **問題：** 若能顯示「全球 28% 的人和你一樣是肯定的言語型」，可強化共鳴感與社群歸屬。
- **建議：** 加入靜態數據卡（使用假設性研究數據，加上資料來源標注）。

## ✅ 目前做得好的地方

- **hreflang 多語言 SEO** 設定正確，5 個語言版本皆有完整標記
- **JSON-LD 結構化資料**（WebApplication + FAQPage）所有頁面均完整
- **Google Fonts** 使用 `display=swap` + preconnect，不阻塞渲染
- **Quiz 重測功能**（`retakeQuiz()`）答案陣列正確重置
- **Cookie 同意橫幅** 含 localStorage 追蹤，符合 GDPR 基本要求
- **聯盟連結** 均使用 `rel="noopener" target="_blank"`
- **RWD 響應式設計** 使用 `clamp()` 與 CSS Grid，移動端友好
- **CNAME + sitemap.xml + robots.txt** 配置完整
- **FAQ 區塊** 有 aria-expanded 無障礙標記，且在 JSON-LD 中亦有標記
- **角色卡片與守護靈** 成功為測驗增加遊戲化與情感連結
